## 1. 一句话定位

**题域官网 = 用精选公开包让人「刷几道题就懂题域」，同时讲清一条龙生态里「练」这一环，并为上游付费服务引流。**

---

## 2. 核心决策（已锁定）

| 决策项 | 选择 | 说明 |
|--------|------|------|
| **主转化** | 先「逛公开包」 | 官网首先是**产品体验入口**，不是下载页 |
| **主叙事** | 一条龙生态里的刷题 App | 首页必须讲清**业务流程与收益链** |
| **体验方式** | 精选题目代替空讲功能 | 公开包 = Demo + 说明书 + 信任状 |
| **内容原则** | 题少、质高、需重规划 | **内容策略优先于 UI 打磨** |
| **主访客** | 备考自学者 + 对生态感兴趣的路人 | 作者/运维不是 v1 官网主受众 |

### 2.1 主 CTA 与叙事（不可漂移）

- **Hero 主按钮**：`体验官方精选题库` → `/packs`（P1 起支持深链 `/packs?featured=1`）
- **Hero 主标题方向**：一条龙备考工具链里的**刷题与共享层**
- **Hero 副标题方向**：先刷官方精选题库，再决定是否导入自己的 Excel
- **次 CTA**：`了解一条龙服务` → 首页 `#ecosystem` 锚点
- **第三 CTA**：`下载 App` → `/download`（降级，非首屏唯一目标）

**首页改版分两档（已拍板）**

| 档位 | 内容 | 是否依赖 featured 包 |
|------|------|----------------------|
| **P0-a** | Hero CTA 层级 + `#ecosystem` 锚点与文案 | **否**，可立即实施 |
| **P0-b / P1** | 首页「精选包」卡片区（Supabase 真实数据） | **是**，需至少 1 个体验包上架 |

### 2.2 明确不做（v1）

- Web 端刷题（完整练习仅在 App）
- Web 端答题交互（预览只读，不记分、不判题）
- 用户注册 / 登录
- `/admin` 管理台
- 无筛选的全量公开包目录当首页体验（会稀释「高质量体验」）
- 像素级模仿 Notion 或先做复杂动效

## 3. 商业与生态叙事

### 3.1 收益链（官网必须讲清）

```
PDF 可搜索化 → DeepTutor 部署与定制出题 → Excel 导出补丁 → 题域刷题 → 公开包订阅
     ↑ 主要收益在前段                          ↑ 题域是「练」     ↑ 后期订阅收入
```

**主要收益来源（前段）**

1. PDF 转为可复制、可搜索的 PDF  
2. DeepTutor 部署  
3. Excel 导出补丁  
4. 定制 DeepTutor 出题  

**后期收益**

- 公开包订阅（区分**免费官方包**与**付费包**）

### 3.2 官网在收益链中的角色

| 环节 | 官网职责 |
|------|----------|
| 前段服务（PDF / DeepTutor / 定制） | v1：首页 `#ecosystem` 区块说明 + 联系方式；不做复杂报价系统 |
| 题域 App | 体验包引流 → `/download`；不做 Web 版刷题 |
| 公开包 | **核心**：精选体验 → 订阅意愿 → 后期付费包 |

## 4. 用户路径（目标态）

```text
首页：生态故事 + 主按钮「体验官方精选题库」
  ↓
/packs：官方精选区（少量包）+ 可选全量/搜索（次级）
  ↓
/pack/[id]：高质量预览（3 题含选项+解析）+ onboarding
  ↓
/docs：导入与订阅细节（已感兴趣的人再看）
  ↓
/download：想完整刷题时再下载 App
  ↓
（P2）付费包 / 前段定制服务咨询
```

### 4.1 与现状差距（待实施）

| 页面 | 现状 | 目标 |
|------|------|------|
| `/` | 三按钮并列；功能 bullet 为主 | **P0-a**：单一主 CTA + `#ecosystem`；**P1**：嵌入精选包卡片 |
| `/packs` | `limit: 50` 全量列表 | 置顶「官方精选」；按 tag 筛选；无 featured 时见 §5.5 |
| `/pack/[id]` | 前 3 题仅题干 + 下载按钮 | onboarding 固定模块 + **3 题含选项与解析**（只读）+ 正确 CTA 层级 |
| `/docs` | Excel 字段在前 | 「发现/订阅」提前；支撑预览页引流 |
| `/download` | 独立引导 | 保持次级入口，文案与预览页一致 |

## 5. 公开包内容策略（P0，先于大改 UI）

### 5.1 三层包结构

| 层级 | 用途 | 数量建议 | 官网展示 |
|------|------|----------|----------|
| **官方体验包** | 5 分钟内懂题域 | **2～3 个**，首包每包 **10～15 题**（成熟后可扩至 ~30，仍保持「少而精」） | 首页 + `/packs` 置顶 |
| **官方免费包** | 长期信任与留存 | 按学科逐步增加 | 「官方免费」分区 |
| **付费包** | 订阅收入 | 后期上线 | 标价 + 有限预览（v1 官网隐藏分区） |

### 5.2 体验包选题标准

- 每题：清晰题干 + 高质量解析，无凑数题  
- 题型：覆盖单选 / 多选 / 判断等，展示 App 能力  
- 至少 1 包：专门演示「导入 → 练习 → 订阅」叙事（如「题域入门体验」）  
- 题量刻意少：刷完有「还想继续」，而非目录感  
- **Web 预览须能体现解析质量**：见 §5.4 预览内容规范  

### 5.3 Tag 约定（`public_packs.tags`）

短期不改 schema，用 tags 区分：

| Tag | 含义 |
|-----|------|
| `official` | 官方出品 |
| `featured` | 官网首页 / 精选区推荐 |
| `demo` | 入门体验包 |
| `free` | 免费官方包 |
| `paid` | 付费包（后期） |

**组合规则**：首包须同时打 `official` + `featured` + `demo`；同一包可兼多个 tag。

**官网查询优先级**：`featured` → `official` → 全量/search（次级）。

**REST 筛选（PostgREST，`tags` 为 text[]）**

```http
# 精选区
GET /rest/v1/public_packs?tags=cs.{featured}&order=updated_at.desc&limit=10

# 官方免费分区（P1+）
GET /rest/v1/public_packs?tags=cs.{free,official}&order=updated_at.desc
```

实现细节见 `src/lib/supabase.ts`；联调由 `testing-api-tester` Agent 验证。

### 5.4 预览页固定模块（每个 featured 包）

**页面结构（自上而下）**

1. 包类型说明（体验 / 免费 / 付费）  
2. 题域是什么（Android 刷题 + 发现订阅，一句话）  
3. App 内订阅步骤（3 步，链到 `/docs` 锚点）  
4. 题目来源轻量生态条（PDF → DeepTutor → Excel → 题域）  
5. **题目预览（前 3 题）** — 见下方内容规范  
6. CTA：继续逛精选包（主）/ 了解定制服务或下载 App（次）  

**预览内容规范（v1，已拍板）**

| 字段 | 是否展示 | 说明 |
|------|----------|------|
| 题干 | ✅ | 去 Markdown 标记，与 App 预览一致 |
| 选项 | ✅ | 按题型渲染（单选/多选/判断） |
| 正确答案 | ❌ | 不公布，避免 Web 变刷题 |
| 解析 | ✅ | 体现「高质量解析」卖点 |
| 交互 | ❌ | 只读，无答题、判分、下一题 |

题量仍为 **3 题**；完整刷题与订阅导入仅在 App。

### 5.5 零 featured 包时的官网表现

| 位置 | 行为 |
|------|------|
| 首页 Hero / `#ecosystem` | **正常展示**（P0-a 不依赖 Supabase 包数据） |
| 首页「精选包」卡片区 | **整区隐藏**（不占位、不用假数据） |
| `/packs` 精选置顶区 | 隐藏；页顶说明「官方精选题库筹备中」+ 链到 `#ecosystem` 或 `/docs` |
| `/packs` 全量列表 | 仍可展示（次级），或维持搜索；**不作为首页主体验** |
| Hero 主按钮 `/packs` | 保留；用户进入后看到筹备说明而非空卡片 |

## 6. 首页信息架构（改版目标）

### 6.1 区块顺序

1. **Hero** — 生态定位 + 主 CTA「体验官方精选题库」  
2. **#ecosystem** — 一条龙流程图（**与 §3.1 逐步一致**）+ 前段服务一句话 + 联系方式  
3. **精选包** — 2～3 张官方包卡片（真实 Supabase 数据；**无 featured 包时隐藏**，见 §5.5）  
4. **核心能力** — 功能列表（降级，给已感兴趣用户）  
5. **Footer** — 文档 / 下载 / 联系  

**`#ecosystem` 流程步骤（与 §3.1 对齐）**

1. PDF 可搜索化  
2. DeepTutor 部署与定制出题  
3. Excel 导出补丁  
4. 题域刷题（练）  
5. 公开包订阅  

### 6.2 视觉原则（相对 Notion）

**可借鉴**

- 首屏一句话 + 一个主 CTA  
- 留白与区块叙事节奏  
- 文档与营销分离（已有 `/docs`）  

**不照搬**

- SaaS 注册 / Pricing 首屏  
- 功能网格替代 Demo  
- 全站黑白极简（题域需要产品感 + App 截图 + 真实题目）  

### 6.3 全站关键文案（Header / Footer / CTA）

| 位置 | 文案 | 链接 |
|------|------|------|
| Hero 主 CTA | 体验官方精选题库 | `/packs` |
| Hero 次 CTA | 了解一条龙服务 | `/#ecosystem` |
| Hero 第三 CTA | 下载 App | `/download` |
| Header `/packs` | 精选题库（或「发现」暂保留，P1 统一） | `/packs` |
| Footer 产品链 | 精选题库 / 使用文档 / App 下载 | 同上 |

Footer tagline 方向：**先体验精选题库，再导入自己的 Excel**（与 Hero 副标题一致，避免「导入即练」单独抢戏）。

### 6.4 `/packs` 分区顺序（P0 精选区上线后）

1. **官方精选** — `tags` 含 `featured`  
2. **官方免费** — `tags` 含 `free` + `official`（P1+，有数据再显示）  
3. **全部公开包 / 搜索** — 现有全量列表（次级，折叠或置底）  

v1 不展示付费包分区（`paid` tag 预留）。
